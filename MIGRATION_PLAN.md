# ログイン機能完全移植計画

## 現状の問題点
- 認証機能（ログイン・ログアウト）のみ移植済み
- ダッシュボード、マイページ、管理画面が未実装
- 各ロールのblueprint（system_admin, tenant_admin, admin, employee）が未実装

## 移植対象

### 1. Blueprint
- `system_admin.py` (1005行) - システム管理者用
- `tenant_admin.py` (66241行) - テナント管理者用  
- `admin.py` (47431行) - 管理者用
- `employee.py` (7305行) - 従業員用

### 2. テンプレート
#### システム管理者
- `sys_mypage.html` - マイページ
- `system_admin_dashboard.html` - ダッシュボード
- テナント管理画面
- 管理者管理画面

#### テナント管理者
- `tenant_admin_mypage.html` - マイページ
- `tenant_admin_dashboard.html` - ダッシュボード
- 管理者管理画面
- 店舗管理画面

#### 管理者
- `admin_mypage.html` - マイページ
- `admin_dashboard.html` - ダッシュボード

#### 従業員
- `employee_mypage.html` - マイページ
- `employee_dashboard.html` - ダッシュボード

### 3. 機能
- マイページ（プロフィール表示・編集、パスワード変更）
- ダッシュボード（各ロール専用）
- ユーザー管理（作成・編集・削除）
- テナント管理（システム管理者のみ）

## 移植方針
1. 定款作成機能に関係ない機能は除外（アンケート、予約、スタンプカード等）
2. 基本的なユーザー管理機能は全て移植
3. マルチテナント対応は維持
4. 段階的に移植してコミット

## 移植順序
1. システム管理者のマイページとダッシュボード
2. テナント管理者のマイページとダッシュボード
3. 管理者のマイページとダッシュボード
4. 従業員のマイページとダッシュボード
5. ユーザー管理機能
6. テナント管理機能
